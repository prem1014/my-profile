{{"If you are used to use 'var' keyword, you should know javascript hoisting. Whenever we create a variable or object using javascript 'var' keyword, javascript pushes the declaration of that variable or object to top of the current scope. Let us understand this javascript concept by an example."}}

{{"function"}} {{"hoistingExample(){"}}

{{"console.log(x)"}} {{"// undfined"}}

{{"var "}} {{"x = 20;"}}

{{"}"}}

{{"Equivalent to =>"}}

{{"function"}} {{"hoistingExample(){"}}

{{"var "}} {{"x;"}}

{{"console.log(x)"}} {{"// undfined"}}

{{"x = 20;"}}

{{"}"}}

So as per above example, each variable declaration get push to the top of the function. Make a note that only declaration get pushed to top not the initialization. That's why output is undfined

Note: Function expressions are not hoisted. If you create a function using var keyword, that function can't be hoisted.
